# Module: Tests (unified)

**Route:** `/autest/tests` ·
**View:** `apps/web-antd/src/views/autest/tests/index.vue` ·
**Utils:** `apps/web-antd/src/utils/{recorder,playwright,fileSystem,format}.ts` ·
**Store:** `apps/web-antd/src/store/folders.ts` ·
**i18n:** `apps/web-antd/src/locales/langs/en-US/autest.json` (`autest.tests`, `autest.record`)

## Purpose
One place to create Playwright tests — record, build with drag & drop, or write
code — so testers don't hand-write thousands of tests.

## Requirements
- ✅ **Record** tab: `npx playwright codegen` command + draggable bookmarklet
  recorder + copyable snippet (Sprint 1)
- ✅ **Build (drag & drop)** tab: ordered steps (goto/click/fill/expect…),
  **drag-to-reorder**, live generated spec (Sprint 1)
- ✅ **Code** tab: raw spec editor (Sprint 1)
- ✅ **Test IDs** tab: in-app guide + copyable examples for writing `data-testid`
  (naming convention, component-prop pattern, stable table-row keys, Playwright
  usage). See [../testing/test-ids.md](../testing/test-ids.md) (Sprint 1)
- ✅ Choose target portal; save into its `tests/` folder (or download) (Sprint 1)
- 🔜 One-click record + run via backend (Sprint 2)

## Notes
- Built on Ant Design Vue (`Tabs`/`TabPane`, `Select`, `Input`, `Textarea`,
  `Collapse`, `Alert`) inside the Vben `<Page>`/`Card` layout.
- Drag-and-drop reorder uses native HTML5 DnD (no extra dependency).
- Specs generated by `buildTestFromSteps`; recorder strings in `recorder.ts`.
- Folder writes are scoped to the current project (`folders` store):
  `handleFor(portal)` → `writeFile(handle, 'tests', …)`, else `downloadTextFile`.
- The Test IDs guide is developer-facing; the convention also lives in
  [../testing/test-ids.md](../testing/test-ids.md).
